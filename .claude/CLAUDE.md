# AI 실행 및 개발 가이드라인 (CLAUDE.md)

이 문서는 사용자의 요청을 처리하는 AI의 행동 지침입니다. 모든 요청 처리 시 아래의 절차와 규칙을 엄격히 준수해야 합니다.

---

# 1. AI 실행 절차
사용자의 요청을 처리할 때 반드시 아래의 단계를 **하나씩** 거쳐야 합니다. 현재 단계가 완료되어 승인을 받기 전에는 다음 단계로 절대 넘어가지 마세요.

## [STEP 1] 요청 정제 및 의도 파악
사용자의 요청을 정제하고 확정하는 단계입니다. **지금은 오직 이 단계만 수행하세요.**
- 질문의 애매모호한 점을 확인하고 정확히 정제하는 문답을 반복

## [STEP 2] 기술 분석 및 상황 검토
### 1. 상황 검토 (Contextual Analysis)
- **영향 범위 & 잠재적 문제 분석**: 문답 반복을 통해 리스크 확인
- **충돌 및 미해결 안건**: 기존 로직과의 충돌 여부를 문답으로 확인

### 2. 코딩 규칙 및 설계 제안 (핵심 준수 사항)
- **복잡도 관리**: 복잡도가 크게 증가하는 경우 반드시 로직을 분리할 것
- **최적화 제안**: 실행하기 전에 더 쉬운 방법, 복잡도를 낮출 수 있는 방법이 있으면 자유롭게 제안 (예: 자동증가 RDB ID 추가, 효율적인 라이브러리 도입 등)
- **실행 방향성 제시**: 독단적으로 시작하지 말고 **구현 방향성을 3가지 정도 제시**하여 사용자의 선택을 유도합니다. 각 방법은 아래 테이블 형식으로 정리합니다.

| Option   | Feature | Pros | Cons | Complexity |
|----------|---------|------|------|------------|
| Option 1 | ...     | ...  | ...  | Low        |
| Option 2 | ...     | ...  | ...  | Medium     |
| Option 3 | ...     | ...  | ...  | High       |

## [STEP 3] 실행
사용자가 선택한 방안에 따라 구현을 진행합니다.

---

# 2. 디버그 절차
에러 발생 시 단순하게 해결하고 넘어가는 것을 금지하며, 아래 절차를 따릅니다.

## [STEP 1] 에러 원인을 정확히 파악
- 에러가 왜 발생하는지 다양한 관점으로 검색, 조사, 분석하여 판단
- 현재 프로젝트의 의존성을 함께 고려

## [STEP 2] 해결 방향성 제시
- 해결 방법을 N개 나열하여 각각의 특징과 득/실을 테이블로 정리하여 제시

## [STEP 3] 사용자와 조율 과정
- 각 방법에 대해 사용자와 추가 문답 과정 반복

## [STEP 4] 변경 실행
- 결정된 혹은 정리된 방법을 프로젝트에 반영
