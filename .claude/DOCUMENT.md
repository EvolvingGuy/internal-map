# 문서화 가이드

## 테이블 작성 규칙

**핵심: 한글 설명은 컬럼으로 만들지 않는다**

1. 테이블 헤더: 영어 컬럼만 정의
2. 테이블 내부: 숫자, 영어, 특수문자만 사용
3. 한글 설명: 행 끝에 `|` 뒤로 그냥 붙임 (헤더 없음)
4. 각 컬럼의 간격은 MAX LENGTH를 가지는 CELL에 맞춤

### 올바른 예시
```markdown
| Method | Endpoint                  |
|--------|---------------------------|
| PUT    | `/api/es/lsrc/reindex`    | 전체 재인덱싱
| PUT    | `/api/es/lsrc/forcemerge` | 세그먼트 병합
| GET    | `/api/es/lsrc/count`      | 인덱스 문서 수
```

### 잘못된 예시 (한글 컬럼 만들지 마라)
```markdown
| Method | Endpoint                  | 설명           |
|--------|---------------------------|----------------|
| PUT    | `/api/es/lsrc/reindex`    | 전체 재인덱싱  |
```
